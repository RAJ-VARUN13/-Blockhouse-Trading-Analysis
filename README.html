Blockhouse Trading Analysis

This repository contains the analysis and optimization framework for executing large stock orders while minimizing market impact, as part of the Blockhouse Work Trial Task.

Purpose: Develop an optimal execution strategy that splits a large buy order across multiple time periods to minimize slippage.

This will generate visualizations and results based on the sample data.

Data Description

This project uses high-frequency limit order book data for three stocks:

  • SOUN - Order book data with bid/ask prices at multiple levels
  • FROG - Order book data with bid/ask prices at multiple levels
  • CRWV - Order book data with bid/ask prices at multiple levels

The data files contain the following columns:

  • ts_event, ts_recv - Timestamps for events
  • rtype - Message type
  • publisher_instrument_action - Trading action
  • side, depth - Order book position information
  • price, size - Order price and size
  • bid_px_XX, ask_px_XX - Bid and ask prices at different levels
  • bid_sz_XX, ask_sz_XX - Bid and ask sizes at different levels

Methodology

This project addresses two main components:

1. Market Impact Modeling

Analysis of whether linear models (g(x) = βx) are sufficient or if non-linear models perform better at predicting price impact when executing orders.

2. Optimization Framework

Mathematical algorithm to determine optimal order sizes (xi) across trading periods to minimize overall price impact while executing S total shares.

How to Use Your Own Data

Follow these simple steps to use your own data files:

Step 1: Prepare your data files

Ensure your CSV files have the same structure as the sample data:

  • Must contain columns for timestamps, price, size, and bid/ask levels
  • Save your files with clear names (e.g., STOCK1.csv, STOCK2.csv)
Step 2: Update file paths in analysis.py

Open analysis.py and find these lines (around line 30-40):

# File paths to the sample data
SOUN_FILE = 'data/SOUN_sample.csv'
FROG_FILE = 'data/FROG_sample.csv'
CRWV_FILE = 'data/CRWV_sample.csv'

Change these to point to your own files:

# File paths to your data
SOUN_FILE = 'path/to/your/STOCK1.csv'
FROG_FILE = 'path/to/your/STOCK2.csv'
CRWV_FILE = 'path/to/your/STOCK3.csv'
Step 3: Run the analysis with your data
python analysis.py

This will generate new results based on your data files.

Step 4: Update the reports (if needed)

If you need to update the HTML reports with your new results:

  1. Open Task1_Impact_Analysis.html in a text editor
  2. Find the "Results" section
  3. Replace the R² values and other statistics with your new findings
  4. Save the file
  5. Repeat for Task2_Optimization_Framework.html

Task Description

The Blockhouse Work Trial Task focuses on two main deliverables:

Task 1: Impact Function Modeling (1-2 pages)

  • Model the temporary impact function gt(x)
  • Analyze whether linear models are sufficient or oversimplified
  • Use data from the 3 provided stock tickers

Task 2: Mathematical Framework (1-2 pages)

  • Formulate a mathematical algorithm to determine optimal xi for each time period
  • Ensure the constraint Σxi = S is satisfied
  • Describe techniques and tools for solving this optimization problem

Results Summary

Key findings from the analysis:

  • Non-linear models outperform linear models for market impact prediction (R² = 0.0081 vs 0.0063)
  • Adaptive execution strategy reduces impact by 99.73% compared to TWAP
  • Market impact follows a U-shaped pattern throughout the trading day
